Tell the developer:

Build Kontra as a compliance-first loan servicing operating system, not just a dashboard. Start by creating a centralized loan record for each asset that stores borrower, property, loan terms, payment status, reserves, insurance, taxes, rent rolls, financials, inspections, covenants, watchlist status, default status, documents, approvals, and full audit history. Then build an event-based workflow engine where every servicing issue opens a structured case, including Watchlist, Hazard Loss, Draw Request, Property Management Change, Inspection Repair, Covenant Breach, Default/ROR, and Tokenization Eligibility. Each case must include required document checklists, due dates, responsible parties, comments, approval steps, status tracking, escalation logic, and closeout requirements.

Add an admin-controlled rules engine so Kontra can update servicing requirements without code changes. The admin should be able to configure DSCR triggers, occupancy triggers, repair classifications, required documents, reserve requirements, default interest logic, inspection frequency, approval authority, escalation timelines, and tokenization eligibility rules. AI should be used only to assist with document review, data extraction, risk flagging, and workflow recommendations, but not to automatically approve servicing decisions. Every AI output must show the source document, confidence score, reason, and require human approval before action.

For tokenization, build it as a separate permissioned layer that only activates after a loan passes servicing compliance checks. A loan or pool should not be eligible for tokenization unless taxes, insurance, financials, rent roll, reserves, inspections, critical repairs, defaults, legal documents, and investor disclosures are complete and current. Add KYC/AML, investor accreditation/eligibility, transfer restrictions, ownership ledger, investor dashboard, distribution tracking, risk updates, and audit-ready disclosure history. The final architecture should make servicing the source of truth, compliance the gatekeeper, and tokenization the downstream financial product.
